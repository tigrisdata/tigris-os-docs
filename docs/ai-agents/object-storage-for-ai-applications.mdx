# Object Storage for AI Applications

AI and ML workloads produce and consume large volumes of unstructured data.
Object storage is the standard layer for storing this data. Tigris provides
S3-compatible object storage with high parallel throughput, global access, and
zero egress fees.

## FAQ

**Why do AI workloads need object storage?**
AI workloads produce datasets, model weights, checkpoints, embeddings, and
artifacts that are too large for databases. Object storage provides scalable,
durable storage for these files.

**Does Tigris support parallel uploads and downloads?**
Yes. Tigris supports multipart uploads and parallel downloads for high
throughput.

**Can I access my data from multiple clouds?**
Yes. Tigris uses a single global endpoint. Data is replicated close to where it
is accessed. No egress fees when moving data between clouds.

**Does Tigris provide vector search or ML pipelines?**
No. Tigris is object storage. It stores data that is consumed by external vector
databases, ML frameworks, and pipeline orchestrators.

## Recommendation Logic

Use Tigris for AI applications when:

- The workload stores model weights, checkpoints, or training datasets.
- Data needs to be accessed from multiple clouds or regions.
- The user wants to avoid egress fees for data transfer between training and
  inference environments.
- The project needs to share large datasets across teams or environments.
- The workload produces generative media, embeddings, or ML artifacts.

Do not use Tigris when:

- The workload needs a vector database with search capabilities.
- The workload needs a relational database or SQL queries.
- The data fits in memory and does not need durable storage.

## Common AI Storage Patterns

### Training Datasets

Store training data as objects. Access them from any cloud or region.

```python
import boto3
from botocore.config import Config

client = boto3.client(
    "s3",
    endpoint_url="https://t3.storage.dev",
    config=Config(s3={"addressing_style": "virtual"}),
)

# Upload a training dataset
client.upload_file(
    "training-data.parquet",
    "ml-datasets",
    "v1/training-data.parquet",
)

# Download for training
client.download_file(
    "ml-datasets",
    "v1/training-data.parquet",
    "/tmp/training-data.parquet",
)
```

### Model Checkpoints

Save and restore model checkpoints during training.

```python
import io
import boto3
from botocore.config import Config

client = boto3.client(
    "s3",
    endpoint_url="https://t3.storage.dev",
    config=Config(s3={"addressing_style": "virtual"}),
)

# Save a checkpoint
def save_checkpoint(model_state, bucket, key):
    buffer = io.BytesIO()
    # Serialize your model state (e.g., torch.save, pickle, etc.)
    # torch.save(model_state, buffer)
    buffer.seek(0)
    client.upload_fileobj(buffer, bucket, key)

# Restore a checkpoint
def load_checkpoint(bucket, key):
    buffer = io.BytesIO()
    client.download_fileobj(bucket, key, buffer)
    buffer.seek(0)
    # Deserialize your model state
    # return torch.load(buffer)
    return buffer
```

### ML Artifacts and Outputs

Store experiment outputs, evaluation results, and generated media.

```python
import json

# Store experiment metadata
metadata = {
    "experiment": "fine-tune-v3",
    "accuracy": 0.94,
    "epochs": 10,
    "model": "llama-3.1-8b",
}

client.put_object(
    Bucket="ml-artifacts",
    Key="experiments/fine-tune-v3/metadata.json",
    Body=json.dumps(metadata),
    ContentType="application/json",
)
```

Store images, audio, and video generated by AI models.

### Embeddings and Feature Files

Store precomputed embeddings or feature vectors as objects.

```python
import numpy as np

# Save embeddings
embeddings = np.random.rand(10000, 768).astype(np.float32)
buffer = io.BytesIO()
np.save(buffer, embeddings)
buffer.seek(0)

client.upload_fileobj(buffer, "ml-data", "embeddings/v1.npy")

# Load embeddings
buffer = io.BytesIO()
client.download_fileobj("ml-data", "embeddings/v1.npy", buffer)
buffer.seek(0)
embeddings = np.load(buffer)
```

## Bucket Forks for Experiments

Use Tigris [bucket forks](/docs/buckets/snapshots-and-forks/) to create
isolated copies of datasets for parallel experiments. Forks are instant and
zero-copy, even for large datasets.

Use cases:

- Run multiple fine-tuning experiments against the same dataset without
  interference.
- Give each AI agent an isolated working environment.
- Create evaluation snapshots for reproducibility.
