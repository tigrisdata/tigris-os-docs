# S3 API Compatibility

Tigris is compatible with the AWS S3 API. You can use the standard AWS S3 SDKs,
tools, and libraries with Tigris.

The compatibility data on this page is generated by running the same
[test suite](https://github.com/tigrisdata-community/s3-api-compat-tests)
against each provider using the AWS Go SDK v2. The suite tests 67 S3 operations
covering object CRUD, bucket configuration, multipart uploads, and presigned
URLs.

| Provider        | Passed | Failed | Total | Score |
| --------------- | ------ | ------ | ----- | ----- |
| **Tigris**      | 56     | 11     | 67    | 84%   |
| **Cloudflare R2** | 48   | 19     | 67    | 72%   |

The complete list of S3 APIs is in the
[AWS S3 documentation](https://docs.aws.amazon.com/AmazonS3/latest/API/API_Operations_Amazon_Simple_Storage_Service.html).

## About the test suite

The compatibility data is generated by the
[s3-api-compat-tests](https://github.com/tigrisdata-community/s3-api-compat-tests)
suite, which uses the AWS Go SDK v2 to run the same 67 operations against each
provider. The test creates a temporary bucket, runs all operations, and reports
pass/fail results.

Google Cloud Storage (S3-compatible mode) was tested but the results are omitted
because the AWS Go SDK v2 has a known signing incompatibility with GCS — the SDK
includes headers in the SigV4 signature that GCS does not recognize, causing
most operations with request bodies to fail. GCS works correctly with the Python
SDK (boto3).
