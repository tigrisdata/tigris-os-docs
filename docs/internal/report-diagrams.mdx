# Antithesis report — diagram previews

Draft previews of the 7 diagrams for the Tigris Antithesis report. This page is
not linked from any sidebar and should not be merged. The standalone HTML
renders live in `docs/internal/_html/` for offline review.

All diagrams are drafts — please leave comments on the PR pointing at what
should change (copy, layout, emphasis). We'll iterate one by one once feedback
is in.

  <DeleteThenReadRaceDiagram />

**Design notes:**

- 5-lane sequence diagram (Request A, Key Lock, FDB, Edge Cache, Request B) with
  vertical dashed lifelines.
- The race window is a translucent coral band spanning all lanes — makes it
  obvious the cache is "live" during the gap.
- Coral-colored step 3 and 4 (GET and stale return) are the bad interleaving.

## 4. The Fix: Reorder, Evict Before Delete

**Placeholder:** _Before/after sequence. Left ("Before"): FDB delete → deferred
eviction with race window and a concurrent GET sneaking in. Right ("After"):
eager eviction under lock, concurrent GET blocked._

<div className="mermaid-frame">
  <BeforeAfterFixDiagram />
</div>

**Design notes:**

- Two zoned panels side-by-side: coral dashed border (Before), green dashed
  border (After).
- "Before" shows the race window inline as a coral band between steps 2 and 3;
  "After" replaces it with a green "no window" band.
- Step 2 in the "After" panel (the moved-up eager eviction) gets the green
  accent, calling out the one structural change.
